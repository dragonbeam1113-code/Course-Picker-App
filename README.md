# 履修候補ピッカー

## 概要

履修候補ピッカーは、弘前大学の学生が履修登録時に大量のシラバスを読む前に、自分の目的・興味・条件に合いそうな授業候補を絞り込むためのWebアプリのプロトタイプです。

このプロトタイプでは、添付された弘前大学理工学部のシラバスPDFから作成したサンプル授業データを使い、ユーザーの入力条件と授業タグを照合して、おすすめ授業を3〜5件提示します。

## 想定ユーザー

弘前大学の学生。特に、履修登録時に多くのシラバスを詳細に読んでから履修候補を絞ろうとする学生。

## 解決したい課題

履修登録時、自分が取れる科目を大量に確認し、その中から自分の目的・興味・条件に合う候補を探すのに時間がかかる。

従来の流れは次のようになりがちです。

1. 履修できそうな科目のシラバスを大量に読む
2. 気になった授業をメモする
3. メモした授業をもう一度シラバスで確認する
4. さらに候補を絞る

このプロトタイプは、最初からすべてのシラバスを読むのではなく、事前に入力した判断基準に合いそうな授業候補を提示することで、履修登録時の選択コストを下げることを目指します。

## ユーザーが得る価値

ユーザーは、自分の目的・興味・条件に合いそうな授業を3〜5件提示してもらうことで、最初のシラバス確認の負担を減らせます。

## 検証したい仮説

履修できる可能性のある科目をすべて詳しく読んだうえで履修科目を決めようとする弘前大学の学生に対して、事前に選考基準を入力すると候補授業をいくつか提示してくれるプロトタイプを見せれば、履修登録にかかる時間や選択コストを削減できるという価値を感じるはずである。

## MVPで実現すること

このMVPでは、以下の機能に絞ります。

* 条件入力フォーム
* サンプル授業データの利用
* タグ・条件一致によるスコアリング
* おすすめ授業3〜5件の表示
* おすすめ理由の表示
* 注意点の表示
* 公式シラバスで確認すべき事項の表示
* おすすめされた授業の比較表の表示

## 入力項目

ユーザーは以下を入力または選択できます。

* 学年
* 興味分野
* 履修目的
* 重視する条件
* 避けたい条件
* 自由記述メモ

想定する興味分野の例：

* 数学
* 物理
* プログラミング
* 計算物理
* 量子力学
* 電磁気学
* 代数学
* 解析学
* 数値計算
* 抽象数学

想定する履修目的の例：

* 専門基礎を固めたい
* 物理を深く学びたい
* 数学の基礎を固めたい
* プログラミング・計算系に強くなりたい
* 将来の研究や開発に役立てたい
* 必修科目を優先したい
* 演習で実力をつけたい

想定する重視条件の例：

* 必修
* 演習がある
* プログラミングに関係する
* 物理に関係する
* 数学に関係する
* レポート中心
* 試験中心
* 基礎力がつく
* 応用につながる

想定する避けたい条件の例：

* 課題が重い
* 試験比重が高い
* 発表がある
* 毎回小テストがある
* 前提知識が多い
* 連続コマ
* 4単位科目

## 出力項目

推薦結果として、各授業について以下を表示します。

* 授業名
* 曜日・時限
* 単位数
* 担当教員
* おすすめ度スコア
* おすすめ理由
* 注意点
* 公式シラバスで確認すべき事項
* タグ
* 短い授業概要

また、おすすめされた授業を比較する表も表示します。

比較表には以下を含めます。

* 授業名
* 分野
* 曜日・時限
* 単位数
* 評価方法
* 向いている学生
* 注意点

## サンプル授業データ

このプロトタイプでは、以下の8科目をサンプルデータとして使用します。

* 電磁気学II
* 代数学I
* 量子力学I
* 代数学演習I
* 微分方程式
* 電磁気学演習
* 集合・位相I
* 計算機演習

データはPDFシラバスをもとに、MVP向けに短い要約、タグ、注意点、確認事項として整理したものを使います。

## スコアリング仕様

ユーザーの入力条件と授業データのタグ・概要・推奨対象・注意点を照合し、授業ごとにスコアを算出します。

基本方針：

* 興味分野と授業タグが一致したら加点
* 履修目的と recommendedFor が一致したら加点
* 重視条件と授業タグ・評価方法・授業形式が一致したら加点
* 避けたい条件と授業の注意点が一致したら減点
* 必修を重視する場合、必修科目を加点
* 演習を重視する場合、演習科目を加点
* プログラミングや計算系を重視する場合、計算機演習を高く評価
* 数学基礎を重視する場合、代数学I、代数学演習I、微分方程式、集合・位相Iを評価
* 物理基礎を重視する場合、電磁気学II、電磁気学演習、量子力学Iを評価

スコア上位3〜5件をおすすめ授業として表示します。

## 作らない機能

このMVPでは、以下の機能は実装しません。

* 弘前大学シラバス全件の自動取得
* シラバスサイトのスクレイピング
* 外部LLM API連携
* RAG検索
* ベクトルデータベース
* ログイン
* ユーザーごとの履修履歴保存
* 卒業要件判定
* 時間割の自動生成
* 履修登録可否の断定
* 教務情報との連携
* 口コミ・先輩レビュー機能
* 複数大学対応
* 管理画面

## 注意書き

画面内に以下の注意書きを表示してください。

このアプリは弘前大学の公式サービスではなく、履修候補を整理するための非公式プロトタイプです。授業情報の正確性・最新性は保証しません。履修登録前には、必ず公式シラバス、履修要項、教務情報を確認してください。このアプリは履修可否、卒業要件、単位取得を保証しません。
